# record-event — 事件记录工作流

**触发词**：发生了什么 / 事件记录 / 记一下 / 新事件

**前置条件**：用户描述了一个事件或经历，需要记录到 L1。

---

## 流程

### Step 1：提取事件要素

向用户确认以下要素（如果信息不全）：

- **事件标题**：一句话描述
- **时间/背景**：何时何地发生
- **参与者**：谁做了什么
- **决策点/冲突点**：最关键的转折
- **结果**：最终结果如何

### Step 2：生成 L1 记录

```markdown
# 事件记录：[标题]
> 日期：[YYYY-MM-DD]
> 层次：L1

## L1 事件层

**事件标题**：[一句话描述]
**时间/背景**：[何时何地发生]
**参与者**：[谁做了什么]
**核心冲突/决策点**：[最关键的转折]
**最终结果**：[结果如何]

## 待升华
- [ ] 需要补充：从事件中提炼经验（L2）
- [ ] 需要补充：发现跨事件模式（L3）
```

### Step 3：确认存储位置

询问用户存储位置偏好：
- 本地文件（如 `events/YYYY-MM-DD.md`）
- 知识库指定位置
- 或按用户指定路径

### Step 4：写入并验证

1. 写入文件
2. 返回文件路径
3. 询问是否需要继续走 `ascend` 流程

### Step 5：升华触发建议（可选）

记录完成后，根据事件性质给出升华建议：

| 事件类型 | 建议升华路径 | 触发词 |
|---------|-------------|--------|
| 失败/挫折事件 | 优先 L1→L2，提炼教训 | "从这件事学到了什么" |
| 成功/突破事件 | 优先 L2→L3，找到成功规律 | "为什么这次成功了" |
| 反复出现的问题 | L2→L3，识别跨事件模式 | "这已经是第N次了" |
| 重大决策时刻 | L3→L4，提炼决策原则 | "以后遇到类似情况应该怎么做" |
| 身份认知变化 | L4→L5，连接核心身份 | "我想清楚我是谁了" |
