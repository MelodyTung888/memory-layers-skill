# report — 记忆报告生成

**触发词**：生成记忆报告 / 给我讲讲我的模式 / 我的智慧系统 / 记忆总结

**前置条件**：已有至少 3 条以上 L2-L4 的内容。

---

## 报告类型

| 类型 | 触发条件 | 内容重点 |
|------|----------|----------|
| 模式报告 | 用户想了解自己的规律 | L3 模式 + 证据链 |
| 原则报告 | 用户想审视自己的准则 | L4 原则 + 来源 + 反例 |
| 身份报告 | 用户想清晰定位自己 | L5 身份 + 支撑关系 |
| 完整报告 | 用户想全面了解系统 | 五层完整梳理 |

---

## 模式报告格式

```markdown
# 你的模式报告

## 已识别的模式

[按置信度排序，列出所有模式]

### 模式 1：[名称]
- **描述**：[一句话规律]
- **支持证据**：[L2 经验列表]
- **适用边界**：[什么时候成立]
- **反例**：[什么时候不成立]

## 模式间的关联

[不同模式之间的关系，如有]

## 置信度分布
- 高置信度：N 个
- 中置信度：N 个（需要补充 X）
- 低置信度：N 个（需要补充 Y）
```

---

## 原则报告格式

```markdown
# 你的原则报告

## 核心原则

[按领域分组，列出所有原则]

### 领域 1：[领域名]
- **P1**：[原则内容]
  - 来源：[来自哪个模式]
  - 反例：[什么时候不适用]

## 原则质量评估

- ✅ 有来源模式支撑：N 条
- ⚠️ 无明确来源：N 条（建议补充证据）
- ✅ 有反例标注：N 条
- ⚠️ 缺少反例：N 条（原则可信度存疑）

## 矛盾检测

[检查原则之间是否有矛盾]

## 建议

[如何增强原则体系]
```

## 身份报告格式

```markdown
# 你的身份报告

## 核心身份

**[一句话身份陈述]**

## 价值主张

**[你为他人/世界带来什么]**

## 支撑路径

```
身份定位
  ├── 价值主张 1
  │     └── 支撑原则 P1, P2
  └── 价值主张 2
        └── 支撑原则 P3, P4
```

## 与过去相比的变化

[身份陈述的演变历史，如有]

## 建议

[如何更清晰地活出这个身份]
```

## 完整报告格式

```markdown
# 五层记忆系统完整报告
> 生成日期：[YYYY-MM-DD]

## L1 事件层概览
- 事件总数：N
- 最近更新：[日期]

## L2 经验层概览
- 有效行动：N
- 失败教训：N
- 关键洞察：N

## L3 模式层概览
- 高置信度模式：N
- 中低置信度模式：N

## L4 原则层概览
- 原则总数：N
- 有来源支撑：N
- 有反例标注：N

## L5 身份层概览
- 核心身份：[一句话]
- 价值主张：[一句话]

## 证据链完整性检查

- [ ] 每条 L4 原则都有 L3 模式支撑
- [ ] 每条 L3 模式都有 L2 经验支撑
- [ ] 每条 L2 经验都有 L1 事件支撑
- [ ] L5 身份有 L4 原则支撑

## 待完善项

- [ ] [具体项]
```

---

## 生成报告前的检查

1. **数据充足性**：至少需要 L2 有内容，否则报告内容太少
2. **选择报告类型**：根据用户需求选择合适类型
3. **突出重点**：不要平铺直叙，要有分析和洞察
