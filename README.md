# Memory Layers — 五层记忆法

[English](./README_en.md) | 中文

把每一次经历，变成持续增值的智慧资产。

## 什么是五层记忆法

一个将零散经历逐步升华的智慧系统，从具体事件出发，逐层提炼出可指导行动的原则和核心身份定位：

| 层次 | 名称 | 问题 | 输出 |
|------|------|------|------|
| L1 | 事件层 | 发生了什么？ | 原始记录 |
| L2 | 经验层 | 学到了什么？ | 教训与洞察 |
| L3 | 模式层 | 有什么规律？ | 跨事件模式 |
| L4 | 原则层 | 指导原则是什么？ | 行动准则 |
| L5 | 身份层 | 我是谁？ | 核心身份定位 |

**核心理念**：知识必须被编译一次，然后持续维护。不是每次重新推导，而是逐层升华。

## v2.0 新增功能（2026-04-10）

本次重大升级新增四大核心能力：

1. **全自动沉淀（auto-distill）** — 对话结束时 AI 自动判断是否值得沉淀，自动提炼并保存，零负担
2. **FTS5 全文搜索** — 跨记忆快速检索，不再大海捞针
3. **周期性提醒机制** — 每隔一段时间自动提醒回看，验证记忆是否仍然成立
4. **技能自生成** — 同一条经验被引用 5 次时，AI 主动提议生成可复用 skill

## 与 Hermes Agent 的对比

| 维度 | Hermes Agent | 五层记忆法 |
|------|-------------|-----------|
| **设计目标** | Agent 的记忆存储与检索 | 经历的逐层蒸馏与升华 |
| **核心机制** | 记忆存储 + 搜索 + 技能复用 | 事件→经验→模式→原则→身份的升华管道 |
| **记忆类型** | 程序化记忆（skills） | 编译后的智慧资产（原则） |
| **自动化程度** | 需手动触发 | **全自动沉淀**，AI 自主判断并保存 |
| **独特价值** | 多平台接入、跨会话搜索 | **知识编译器**：让记忆变成可指导行动的原则 |

**为什么五层记忆法更优？**

Hermes 的 skills 本质是"程序化记忆单元"，解决了"记什么"，但没有解决"记忆如何变成智慧"。五层记忆法的独特价值在于**知识编译器**角色：它不只是存储，更是一套**从具体经历中提炼抽象原则**的升华管道。Hermes 告诉你"记住这件事"，五层记忆告诉你"从这件事中学到了什么原则"。

## 工作流

| 触发词 | 工作流 | 说明 |
|--------|--------|------|
| 五层沉淀、逐层提炼、完整走一遍 | `distill` | 完整走完 L1→L5 |
| 发生了什么、事件记录、记一下 | `record-event` | 仅记录 L1 |
| 从事件提炼经验、从经验找规律 | `ascend` | 单层升华 |
| 检查记忆系统、健康检查 | `status` | 系统现状 |
| 生成记忆报告、给我讲讲我的模式 | `report` | 记忆报告 |
| （无关键词，有值得沉淀的内容） | `auto-distill` | **自动沉淀** |

## 核心约束（红线）

1. **禁止跳过层次**：L2 经验不能凭空产生，必须有具体事件支撑
2. **禁止放之四海而准**：每条原则必须有来源事件或模式支撑
3. **禁止一次性完成**：五层是持续迭代的系统，不是一次性写作
4. **禁止脱离具体上下文**：原则必须绑定到具体领域/场景
5. **禁止 L4 原则复述 L3 模式**：原则必须比模式更具体、更有可操作性
6. **禁止不保存**：识别到值得沉淀的内容时，必须写入 vault
7. **禁止不索引**：每次保存后自动索引到搜索数据库

```

│   ├── L2-experiences/   # 经验层
│   ├── L3-patterns/       # 模式层
│   ├── L4-principles/     # 原则层
│   ├── L5-identity/       # 身份层
│   └── .index.json        # 全局索引
├── search/                # FTS5 搜索数据库
├── reminders/             # 周期性提醒
└── inbox/                 # 随手 inbox
```

## 示例

输入：产品发布会让团队士气低落的事件

输出五层结构：

```
L1: 事件 → 上周产品发布会后团队士气低落
L2: 经验 → 需求变更未及时同步导致返工，沟通要前置
L3: 模式 → 跨部门协作中，信息同步的时效性决定执行质量
L4: 原则 → 任何跨团队依赖的变更，必须在 24h 内同步所有相关方
L5: 身份 → 我是一个重视信息透明度、主动驱动的协作者
```
