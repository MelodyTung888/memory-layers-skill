# Memory Layers — 五层记忆法

> 把每一次经历，变成持续增值的智慧资产。

---

## 首次使用配置

**新用户首次使用时，询问：**

```
请告诉我你要存放记忆的文件夹路径，例如：~/记忆库 或 ~/Desktop/我的记忆

（这个文件夹会由系统自动创建，你只需确认好保存位置、同时给它起个名字）
```

**其余全部用默认值，无需询问：**

- 提醒周期：L1→L2: 7天，L2→L3: 14天，L3→L4: 30天，L2回看: 30天
- 引用阈值：5次（达到此次数时 AI 会通知你准备生成新 skill）
- 脚本目录：内嵌在 skill 里，你无感知

**将配置写入 vault 的 `.config.json`，后续自动读取。**

---

## 核心配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `vault_path` | 记忆 vault 路径 | 用户提供 |
| `scripts_path` | 搜索脚本目录 | 用户提供 |
| `ref_threshold` | 触发 skill 生成的引用次数 | 5 |
| `reminder_days.L1-L2` | L1→L2 提醒周期 | 7 天 |
| `reminder_days.L2-L3` | L2→L3 提醒周期 | 14 天 |
| `reminder_days.L3-L4` | L3→L4 提醒周期 | 30 天 |
| `reminder_days.L2-review` | L2 回看周期 | 30 天 |

## 五层结构

| 层次 | 名称 | 问题 | 输出 |
|------|------|------|------|
| L1 | 事件层 | 发生了什么？ | 原始记录 |
| L2 | 经验层 | 学到了什么？ | 教训与洞察 |
| L3 | 模式层 | 有什么规律？ | 跨事件模式 |
| L4 | 原则层 | 指导原则是什么？ | 行动准则 |
| L5 | 身份层 | 我是谁？ | 核心身份定位 |

**核心理念**：知识必须被编译一次，然后持续维护。不是每次重新推导，而是逐层升华。

## 自动触发规则

### 何时触发沉淀

以下情况**自动触发**沉淀（无需用户要求）：

1. **对话结束时**：发现用户做了重要决策、踩了坑、有突破性洞察
2. **任务完成后**：成功或失败，都值得记录
3. **模式识别时**：发现当前事件与历史模式有共鸣
4. **原则冲突时**：实践中发现原有原则不适用

### 触发判断

- 用户明确说了"沉淀"、"总结" → 直接走 distill
- 用户只是闲聊 → 不触发
- **重要判断**：拿不准是否值得沉淀时，**默认沉淀**（宁可多，不可漏）

## 工作流路由

| 用户意图关键词 | 工作流 |
|---|---|
| "五层沉淀"、"逐层提炼"、"完整走一遍" | → **distill** |
| "发生了什么"、"事件记录"、"记一下" | → **record-event** |
| "从事件提炼经验"、"从经验找规律" | → **ascend** |

| "生成记忆报告"、"给我讲讲我的模式" | → **report** |
| （无关键词，对话中有值得沉淀的内容） | → **auto-distill** |

## 执行约束（红线）

1. **禁止跳过层次**：L2 经验不能凭空产生，必须有具体事件支撑
2. **禁止放之四海而准**：每条原则必须有来源事件或模式支撑
3. **禁止一次性完成**：五层是持续迭代的系统，不是一次性写作
4. **禁止脱离具体上下文**：原则必须绑定到具体领域/场景
5. **禁止 L4 原则复述 L3 模式**：原则必须比模式更具体、更有可操作性
6. **禁止不保存**：识别到值得沉淀的内容，必须写入 vault，不能只存在上下文里
7. **禁止不索引**：每次保存后自动索引到 FTS5 数据库

## FTS5 搜索系统

**搜索脚本**: `{scripts_path}/memory-layers-search.sh`
**索引脚本**: `{scripts_path}/memory-layers-index.sh`
**数据库**: `{vault_path}/search/memory-search.db`

### 使用方式

```
# 搜索记忆
memory-layers-search.sh "压力 决策"

# 自动索引（在 distill 完成后调用）
memory-layers-index.sh L2 "决策教训" "在压力下做了X决策..." "layers/L2-experiences/xxx.md" "2026-04-10"
```

## 技能自生成机制

**触发阈值**：被引用 N 次时（N 可配置，默认 5 次），自动生成可调用 skill

**技能形态**（待具体讨论）：
- 用户可用：行动清单、检查清单
- AI 可用：可复用的 Agent skill，可在未来任务中调用

**引用追踪（自动）**：
- 每次 L3 模式或 L4 原则被调用/引用时，**自动**更新计数
- 达到阈值时，**自动**告知用户：准备生成新 skill

---

## 提醒机制

| 升华路径 | 提醒周期 |
|----------|----------|
| L1→L2 | 7 天后回看，验证经验是否成立 |
| L2→L3 | 14 天后，验证模式是否被新事件支持 |
| L3→L4 | **自动提炼**，无需等待 |
| L2 经验回看 | 30 天后复盘，是否有新的理解 |

**提醒触发**：每次对话结束时检查是否有到期提醒，主动告知用户。
