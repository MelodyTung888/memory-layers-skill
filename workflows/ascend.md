# ascend — 层级升华工作流

**触发词**：从事件提炼经验 / 从经验找规律 / 升华 / 提炼模式

**前置条件**：已有 L1 事件记录，需要向上升华到 L2/L3。

---

## 流程

### 模式 1：L1 → L2（事件 → 经验）

**问题**：
- 这次事件中，什么做对了？
- 什么做错了？
- 最关键的洞察是什么？

**输出格式**：
```markdown
## L2 经验层

### 有效行动
- [具体行动] → [带来的结果]

### 失败教训
- [具体错误] → [造成的后果]

### 关键洞察
- [一句话洞察]
```

**检查**：
- [ ] 每条经验有对应事件细节支撑
- [ ] 洞察具体到"当 X 时，Y 比 Z 更有效"
- [ ] 没有放之四海的废话

---

### 模式 2：L2 → L3（经验 → 模式）

**⚠️ 前置判断：是否有足够的跨事件证据？**

归纳模式需要**至少 2 个事件**作为证据。如果用户只提供了一个事件：
- 在 L3 模式层标注"置信度：低（待补充）"
- 明确说明需要补充什么事件才能提高置信度
- 在待办中注明"需要第二个类似事件来验证模式"

**问题**：
- 这个经验是否与过去的经验有共同规律？
- 能否跨事件泛化？
- 边界条件是什么？

**输出格式**：
```markdown
## L3 模式层

### 跨事件模式
- **模式名称**：[用词或短语命名]
- **模式描述**：[规律本身，不是事件描述]
- **支持证据**：[哪条 L2 经验支持这个模式]
- **边界条件**：[什么时候这个模式成立]
- **反例**：[什么时候这个模式不适用]
- **置信度**：[高/中/低]
```

**检查**：
- [ ] 模式是从经验归纳，不是先入为主
- [ ] 有明确的边界条件
- [ ] 有真实的反例

---

### 模式 3：L3 → L4（模式 → 原则）

**问题**：
- 基于这个模式，什么是必须遵守的准则？
- 什么时候必须这样做？
- 什么时候可以例外？

**⚠️ 常见错误：L4 原则只是 L3 模式的复述**

模式说"人在压力下决策质量会下降"，原则如果只说"要避免在压力下做决策"，这两句话没有区别。L4 原则必须比 L3 模式**更具体、更可操作**：

| L3 模式（规律） | L4 原则（行动准则） | 差异 |
|----------------|-------------------|------|
| 人在压力下决策质量下降 | 供应商替换决策不得在紧迫感驱动下完成，即便时间紧张，也应完成"最小化尽调"（历史记录 + 至少一个客户参考）再决策 | ✅ 原则具体到"最小化尽调"是什么 |
| 人在压力下决策质量下降 | 要谨慎决策 | ❌ 只是换了词，没有深化 |

**输出格式**：
```markdown
## L4 原则层

### 核心原则
- **原则编号**：[P#]
- **原则内容**：["应该做什么"，具体到可执行的动作]
- **来源模式**：[来自哪个 L3 模式]
- **适用条件**：[什么时候成立]
- **反例**：[什么时候不适用]
```

**检查**：
- [ ] 原则比来源模式的表述更具体、更有可操作性
- [ ] 用正面语言表述（"应该做X"而不是"不要做Y"）
- [ ] 有明确的边界

---

### 模式 4：L4 → L5（原则 → 身份）

**问题**：
- 这些原则揭示了你是什么样的角色？
- 你的核心身份是什么？
- 你为他人带来什么独特价值？

**输出格式**：
```markdown
## L5 身份层

### 身份陈述
- **核心身份**：[你是谁，不是你做什么]
- **价值主张**：[你为他人/世界带来什么]
- **支撑原则**：[哪几条 L4 原则共同指向这个身份]
```

**检查**：
- [ ] 身份陈述不涉及具体职业
- [ ] 价值主张是可验证的行动倾向
- [ ] 支撑原则之间没有矛盾

---

## 升华路径选择

根据用户需求，选择升华路径：

| 用户说 | 走哪条路 |
|--------|----------|
| "从这件事学到了什么" | L1 → L2 |
| "这些经历有什么共同规律" | L2 → L3 |
| "我想出一条准则" | L3 → L4 |
| "我想清楚我是谁了" | L4 → L5 |
| "完整走一遍" | L1 → L2 → L3 → L4 → L5 |
